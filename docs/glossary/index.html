Secure Firewall Glossary

Access Control: A method to define levels of access between networks.

Access Control Policy: A set or collection of rules that define how devices on a network communicate.

Access Control Rules: Individual entries within an Access Control Policy that uses matching criteria to control network communication.

Advanced Malware Protection (AMP): Advanced capability to detect and block malware and other threats contained within files traversing networks. Available as a licensed subscription.

Application Visibility & Control (AVC): Allows Secure Firewall to control connections based on the applications used rather than IP, ports, and protocols.

Authentication: A process by which someone or something provides proof that they can access a particular resource. A typical example is a username and password.

Base License: Included with all Firepower Threat Defense firewall products.

Default Action: The final rule within the access control policy used to determine the action applied to traffic that does not matches any rules within the access control policy.

Demilitarized Zone (DMZ): A term used in network security to identify a semi-trusted network. Typically between an internal (trusted) network and an external (untrusted) network.

Device: A term used to referto a managed Cisco Firepower Threat Defense (FTD) firewall, physical or virtual, with the Firepower Management Center (FMC). It is also used more generically to identify network-connected equipment, such as desktop computers, laptops, phones and other equipment.

Encryption: A method to obfuscate the contents of a file or network connection to prevent unauthorized users from seeing the details.

Entitlements: The number and type of licenses you have for products that you own.

Fastpath: An action within a prefilter policy that allows traffic forwarding through the firewall without applying further inspections.

Firepower Chassis Manager (FCM): The user interface used to manage Firepower chassis running FXOS.

Firepower Management Center (FMC): Cisco's centralized management platform used to manage multiple Firepower devices.

Firepower Threat Defense (FTD): The software from Cisco that is deployed onto Firepower hardware, ASA hardware, or virtually in private or public cloud environments.

Firepower eXtensible Operating System (FXOS): The operating system used by Firepower 4100/9300 hardware appliances.

Malware: A piece of software designed to cause harm to computers or obtain personal information illegally. Also, the term given to the license within Firepower that enables Advanced Malware Protection and file control.

Management Information Base (MIB): A collection of Object IDs (OIDs) that describe and match components of a monitored network device. For example, network interface statistics or CPU utilization.

Multi-Instance: A method to logically distribute the resources of a single physical chassis to run multiple firewalls.

Management Station/Host: The device/server that runs the SNMP management software.

Network Analysis Policy (NAP): A policy that governs how traffic is decoded and preprocessed for further evaluation, especially for anomalous traffic that might signal an intrusion attempt.

Network Address Translation (NAT): A function of IP networking that allows private network address spaces to be translated to publicly routable address spaces.

Network Object: A meaningful identifier for an IP address, Subnet or network range used within a policy.

Next Generation Intrusion Prevention System (NGIPS or IPS): Advanced capability to detect and block malicious activity within network communications, typically using signatures to identify threats. An intrusion policy uses intrusion and preprocessor rules (sometimes referred to collectively as intrusion rules) to examine decoded packets for attacks based on patterns. Intrusion policies are paired with variable sets, which uses named values to reflect a network environment accurately. Available as a licensed subscription.

Object Identifier (OID): An address used to uniquely identify managed devices, their components, and their statuses.

Poll: A query issued by the SNMP server towards a network device, usually at periodic intervals.

Prefilter Policy: An initial policy used to quickly determine if traffic should be fastpathed, blocked or sent for further inspection via the access control policy.

Security Intelligence: A database of known malicious entities that provides the first layer of security in a firewall deployment.

Security Module: A modular processing unit installed within a Firepower 9300 chassis (max 3) that runs Firepower Threat Defense (FTD) or ASA software.

Smart Account: A user account that provides access to your organization's license repository.

Smart Licensing: A cloud-based licensing portal from Cisco that allows central and delegated management of product licenses.

Simple Network Management Protocol (SNMP): Used to monitor network devices by allowing a network management station (Server) to send queries to or receive events from network devices.

Subscription: Additional features available for the firewall via optional license purchases.

Threat: A Malicious entity or object that can cause harm to network devices. Also, the term given to the license within Firepower that enables Next Generation Intrusion Prevention (NGIPS).

Token: A unique code that allows products to register to the Smart Licensing portal to retrieve licenses.

Trap: An event notification generated by a network device and sent to the configured SNMP server(s).

Universal Resource Locator (URL): The text you type in when attempting to access a web-based resource. For example: www.cisco.com/support

URL Filtering: Web-based category filtering allows organizations to control user web connections. Available as a licensed subscription.

Variable Set: Variables in a set represent values commonly used in intrusion rules to identify the source and destination IP addresses and ports. Variable sets provide greater detection accuracy and reduce the amount of noise. Whenever the system uses an intrusion policy to evaluate traffic, it uses an associated variable set.

Virtual Account: A way to organize licenses within your organization to provide the ability to structure based on geographical location, departments, or business functions.

Zone: A meaningful identifier that maps to an individual or group of network interfaces attached to a managed firewall used as either sources or destinations in access control rules.
